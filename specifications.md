# SME 마케팅 페이지 퍼블리싱 서비스 - 명세서

## 1. 프로젝트 개요
**목표**: 중소기업(SME)이 마케팅 랜딩 페이지를 쉽게 생성하고 게시할 수 있는 웹 플랫폼 구축.
**타겟 사용자**: 코딩 경험이 거의 없는 비즈니스 소유자.
**비즈니스 모델**: SaaS (Software as a Service) - 사용자가 마케팅 페이지를 게시하고 호스팅하는 비용을 지불.

## 2. 핵심 기능

### 2.1. 서비스 랜딩 페이지 (The Service)
- **히어로 섹션**: 가치 제안 ("몇 분 만에 비즈니스 사이트 만들기").
- **기능 소개**: 템플릿, 사용 편의성 등 소개.
- **Call to Action**: "무료로 시작하기" / "대시보드".

### 2.2. 사용자 대시보드 (The App)
- **사이트 관리**: 생성된 사이트 목록.
- **에디터**: 사이트 콘텐츠를 업데이트하는 간단한 폼 기반 에디터:
    - 회사 소개 (Company Intro)
    - 사업 내용 (Business Content)
    - 상품 소개 (Product Intro)
    - 위치 정보 (Location)
    - 연락처 (Contact)
    - 연락처 (Contact)
    - 게시판 (Board - 공지사항/뉴스)
- **레이아웃 관리**: 섹션 순서 변경 및 표시 여부 설정 (Dynamic Layout).
- **미리보기**: 사이트 실시간 미리보기.

### 2.3. 게시된 사이트 (The Output)
- **동적 라우팅**: 하위 경로(`service.com/site/my-bakery`)를 통해 사이트 제공.
- **SEO 최적화**: 사용자 입력을 기반으로 메타 태그 서버 사이드 렌더링(SSR).
- **반응형 디자인**: 모바일 우선 템플릿.
- **리드 수집**: 사용자 사이트의 "문의하기" 폼을 통해 DB에 데이터 저장.

## 3. 기술 명세

### 3.1. 아키텍처
- **런타임**: Cloudflare Workers (빠른 글로벌 액세스를 위한 엣지 컴퓨팅).
- **프레임워크**: Hono (경량, 웹 표준 준수).
- **데이터베이스**: Cloudflare D1 (엣지 SQLite).
- **프론트엔드**: Hono의 JSX 또는 문자열 템플릿 리터럴을 사용한 SSR (최대 성능 및 SEO).

### 3.2. 데이터베이스 스키마
- **`users`**: ID, 이메일, 비밀번호 해시, 생성일.
- **`sites`**: ID, 사용자 ID, 슬러그(URL 경로), 제목, 테마 설정(JSON), 생성일.
- **`site_content`**: ID, 사이트 ID, 섹션 타입(히어로, 소개, 사업, 상품, 위치), 콘텐츠(JSON).
- **`posts`**: ID, 사이트 ID, 제목, 내용, 작성자, 생성일 (게시판 기능).
- **`leads`**: ID, 사이트 ID, 고객 이름, 고객 이메일, 메시지, 생성일.

## 4. 디자인 및 UX 가이드라인
- **심미성**: 전문적이고 깔끔하며 신뢰감을 주는 디자인 (블루/슬레이트/화이트 팔레트).
- **성능**: Lighthouse 점수 > 90.
- **단순성**: 에디터는 직관적이어야 함. 복잡한 드래그 앤 드롭 대신 구조화된 폼 사용.

## 5. 검증 기준
- [ ] 사용자가 고유한 슬러그로 새 사이트를 생성할 수 있음.
- [ ] 사용자가 콘텐츠(히어로, 기능 등)를 수정하고 저장할 수 있음.
- [ ] 공개 URL(`/site/:slug`)에서 올바른 콘텐츠가 렌더링됨.
- [ ] 공개 사이트의 문의 폼이 리드 데이터를 DB에 저장함.
